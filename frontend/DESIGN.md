# 前端系统设计

## 目标

前端用于把已完成的 Spring Boot 后端能力组织成可演示的内部任务管理系统。核心路径为：登录、任务列表、任务 CRUD、筛选搜索、状态流转、任务关联资讯、仪表盘、Excel 导出。

## 技术栈

- Vue3 + Vite + TypeScript
- Vue Router：页面路由和登录守卫
- Pinia：用户登录态和后续共享筛选状态
- Axios：统一 API 请求、JWT 注入、未登录处理
- Element Plus：表单、表格、弹窗、标签、分页、消息提示
- ECharts：仪表盘状态分布和完成率图表

## 设计基调

依据 `design-system/icinfo-task-management/MASTER.md`，但为内部运营系统做收敛：

- 页面强调扫描效率和重复操作效率。
- 使用白色工作区、清晰边框、稳定表格和紧凑筛选栏。
- 主色用于关键动作，橙色只作为提醒或导出等辅助动作，不铺满页面。
- 卡片圆角不超过 8px。
- 图标统一使用 `@element-plus/icons-vue`，不使用 emoji。
- 所有可点击元素需要清晰 hover、focus 和 loading 状态。
- 响应式至少覆盖 375px、768px、1024px、1440px。

## 路由

| 路径 | 页面 | 权限 |
| --- | --- | --- |
| `/login` | 登录页 | 公开 |
| `/tasks` | 任务列表、筛选、CRUD、状态流转、导出入口 | 登录 |
| `/dashboard` | 任务统计、图表、临期/逾期提醒 | 登录 |
| `/news` | 独立资讯搜索与刷新 | 登录 |

## API 分层

`src/api/http.ts` 负责：

- `VITE_API_BASE_URL` 或 `/api` 作为基础地址。
- 自动读取 `tm_token` 并注入 `Authorization`。
- 统一处理 `401` 或后端 `code=401`。
- 使用 `unwrap` 解开 `{ code, message, data }`。

业务 API 文件：

- `src/api/auth.ts`
- `src/api/tasks.ts`
- `src/api/news.ts`
- `src/api/dashboard.ts`

## 状态管理

`src/stores/auth.ts` 保存：

- `token`
- `user`
- `isAuthenticated`
- `isMentor`

后续如筛选条件需要跨页面保留，可新增 `src/stores/taskFilters.ts`，但优先保持局部状态，避免过早复杂化。

## 页面结构

### 登录页

- 用户名密码登录。
- 导师 Mock 登录。
- 实习生 Mock 登录。
- 登录成功后跳转 `/tasks`。
- 登录失败展示明确错误。

### 任务列表页

- 顶部筛选栏：状态、负责人、截止日期范围、关键词。
- 视图切换：表格/卡片。
- 操作：新增、编辑、删除、状态切换、导出。
- 分页保留筛选条件。
- 新增/编辑/删除/状态更新后刷新当前查询。

### 任务详情/编辑弹窗

- 左侧任务表单。
- 右侧关联资讯。
- 支持按任务标题刷新资讯。
- 支持手动关键词刷新资讯。
- 资讯失败不关闭弹窗，不影响任务保存。

### 资讯页

- 关键词搜索。
- 手动刷新。
- 分页资讯列表。
- 外部链接新窗口打开。

### 仪表盘

- 统计卡片：待办、进行中、已完成、总数、完成率。
- ECharts 状态分布。
- 临期和逾期任务提醒。
- 导师展示全量统计，实习生展示本人任务统计。

## 交付顺序

1. 登录与鉴权。
2. 任务列表与 CRUD。
3. 筛选搜索与状态流转。
4. 实时资讯与任务关联资讯。
5. 仪表盘。
6. Excel 导出。
7. README、截图和最终验收。

每一步对应 `tasks/prompts.md` 中的一个前端提示词，新会话只处理一个阶段。
