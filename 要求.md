一、项目背景

公司需要一个简单的内部任务管理系统，用于实习生和导师之间分配、跟踪日常任务。要求在“2天（48小时）”内完成一个可运行的MVP版本。

二、实现功能

1. 用户任务管理（Backend + Frontend）

（1）用户登录/注册（可简化，使用JWT或Session，允许Mock登录）。
（2）任务CRUD：创建、查看、编辑、删除任务。
（3）任务状态流转：待办 → 进行中 → 已完成（支持拖拽或按钮切换）。
（4）任务筛选/搜索（按状态、负责人、截止日期）。
（5）获取实时资讯模块（skill构建和应用能力）。

功能描述：
系统需要增加一个“实施资讯”专区或侧边栏/独立页面，能够自动或手动获取外部真实资讯，并与任务关联。

具体要求（必须实现至少以下2条）：
① 调用公开的第三方API或RSS，获取最新相关资讯（例如科技行业动态、编程技术资讯、公司相关新闻等）。
② 在任务详情页或任务表单中，能关联显示相关资讯（例如创建“Java后端”任务时，自动/手动获取最新Spring Boot新闻）。
③ 支持资讯的展示、简单搜索和刷新功能。

2. 核心页面（Vue前端）

（1）登录页。
（2）任务列表页（支持卡片/表格两种视图切换）。
（3）任务详情/编辑弹窗。
（4）个人仪表盘（我的待办、已完成统计）。

3. 额外加分项（可选，但强烈建议尝试，体现设计能力）

（1）任务优先级（高/中/低）+ 颜色区分。
（2）简单统计图表（使用Echarts或Chart.js展示完成率）。
（3）任务截止日期提醒（前端倒计时或后端简单校验）。
（4）导师可查看所有任务，实习生只能看自己的。
（5）导出任务列表为Excel/CSV。

三、技术栈要求（必须使用）

后端： Java + Spring Boot（推荐Spring Boot 3.x + MyBatis-Plus或JPA）。
数据库： MySQL（允许使用H2内存数据库快速开发）。
前端： Vue3（Composition API）+ Vite + Element Plus / Ant Design Vue / Naive UI。
其他： RESTful API、Axios、CORS处理。
允许工具： 任意AI工具（Cursor、Copilot、ChatGPT、Claude、通义千问等）均可使用，但必须在代码中或文档中说明AI辅助的部分。

四、交付物

1. 完整可运行代码（Git仓库链接，最好有README）。
2. 数据库脚本（SQL）。
3. 一份设计文档（1-2页即可，Markdown或PDF），必须包含：
   （1）系统架构图（手绘或工具绘制）。
   （2）数据库表设计（ER图或表结构）。
   （3）主要API接口设计。
   （4）技术选型理由 + AI工具使用心得（用了哪些AI、如何验证AI生成的代码、遇到的坑）。
   （5）遇到的问题及解决思路（具体问题截图）。

六、评测重点

1. 设计思路是否清晰（分层、模块化、前后端分离思想）。
2. 代码结构是否规范（包结构、命名、异常处理）。
3. AI工具使用能力（是否高效使用、是否能发现并修正AI错误）。
4. 功能完整度和UI体验。
5. 文档质量和思考深度。
