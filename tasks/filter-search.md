# 筛选搜索

## 模块目标

实现任务列表的多条件查询能力，支持按状态、负责人、截止日期范围、关键词和分页组合筛选，并在前端提供筛选、重置和刷新体验。

## 输入依据

- `plan.md`：筛选搜索模块要求按状态、负责人、截止日期和关键词检索。
- `plan.md`：任务列表页顶部提供状态、负责人、截止日期、关键词筛选。
- `detailed-design.md`：筛选搜索模块包含组合 SQL、空条件查询、关键词模糊匹配、筛选表单、重置和分页刷新。
- `detailed-design.md`：任务列表 API 参数包含 `status`、`assigneeId`、`dueDateStart`、`dueDateEnd`、`keyword`、`page`、`size`。

## 最小任务 checklist

- [ ] 定义任务查询请求参数对象。
- [ ] 查询参数支持 `status`。
- [ ] 查询参数支持 `assigneeId`。
- [ ] 查询参数支持 `dueDateStart`。
- [ ] 查询参数支持 `dueDateEnd`。
- [ ] 查询参数支持 `keyword`。
- [ ] 查询参数支持 `page` 和 `size`。
- [ ] 后端查询逻辑复用当前用户权限范围。
- [ ] 实现空条件查询，返回当前用户可见任务。
- [ ] 实现状态精确匹配。
- [ ] 实现负责人精确匹配。
- [ ] 实现截止日期开始日期过滤。
- [ ] 实现截止日期结束日期过滤。
- [ ] 实现关键词对标题和描述的模糊匹配。
- [ ] 实现多条件组合查询。
- [ ] 实现分页结果返回。
- [ ] 实现 `GET /api/tasks` 列表接口接收筛选参数。
- [ ] 前端封装任务列表查询 API。
- [ ] 前端任务列表页添加状态筛选控件。
- [ ] 前端任务列表页添加负责人筛选控件。
- [ ] 前端任务列表页添加截止日期范围筛选控件。
- [ ] 前端任务列表页添加关键词输入框。
- [ ] 前端实现搜索按钮，按当前筛选条件请求列表。

- [ ] 前端实现分页控件。

## 验收 checklist

- [ ] 空筛选条件下返回当前用户可见任务。
- [ ] 状态筛选结果正确。
- [ ] 负责人筛选结果正确。
- [ ] 截止日期范围筛选结果正确。
- [ ] 关键词可以匹配标题或描述。
- [ ] 多条件组合筛选结果正确。
- [ ] 实习生筛选结果不会越权展示他人任务。
- [ ] 前端重置后恢复默认列表。
- [ ] 分页切换不会丢失筛选条件。
