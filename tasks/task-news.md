# 资讯关联

## 模块目标

实现任务与资讯的关联查询和刷新能力，在任务详情或编辑弹窗中根据任务标题或手动关键词展示相关资讯。

## 输入依据

- `plan.md`：资讯关联模块要求在任务详情或任务表单中按任务关键词展示相关资讯。
- `plan.md`：任务详情/编辑弹窗展示关联资讯列表，支持按任务标题或手动关键词刷新资讯。
- `detailed-design.md`：资讯关联模块包含任务与资讯关联表写入、重复关联去重、任务详情展示关联资讯。
- `detailed-design.md`：资讯关联 API 包含 `GET /api/tasks/{id}/news` 和 `POST /api/tasks/{id}/news/refresh`。

## 最小任务 checklist

- [ ] 设计任务资讯关联实体或表结构字段：`id`、`taskId`、`newsId`、`createdAt`。
- [ ] 为 `task_id + news_id` 设计唯一约束或去重逻辑。
- [ ] 添加任务关联资讯响应 DTO。
- [ ] 添加任务资讯刷新请求 DTO，支持可选 `keyword`。
- [ ] 添加任务资讯关联数据访问 Mapper 或 Repository。
- [ ] 实现查询任务关联资讯 Service。
- [ ] 查询任务关联资讯前校验当前用户可见该任务。
- [ ] 实现根据任务标题生成默认资讯关键词。
- [ ] 实现手动关键词优先于任务标题的刷新规则。
- [ ] 实现任务关联资讯刷新 Service，复用实时资讯刷新能力。
- [ ] 实现刷新后写入任务与资讯关联关系。
- [ ] 实现重复关联去重。
- [ ] 实现资讯刷新失败时不影响任务详情基础信息。
- [ ] 实现 `GET /api/tasks/{id}/news` 关联资讯查询接口。
- [ ] 实现 `POST /api/tasks/{id}/news/refresh` 关联资讯刷新接口。
- [ ] 前端封装任务关联资讯查询 API。
- [ ] 前端封装任务关联资讯刷新 API。
- [ ] 前端任务详情/编辑弹窗增加关联资讯区域。
- [ ] 前端打开任务详情时加载关联资讯。
- [ ] 前端关联资讯区域展示标题、来源、发布时间和跳转链接。
- [ ] 前端提供按任务标题刷新资讯按钮。
- [ ] 前端提供手动关键词输入与刷新按钮。
- [ ] 前端刷新成功后更新关联资讯列表。
- [ ] 前端刷新失败时只提示资讯错误，不关闭任务弹窗。

## 验收 checklist

- [ ] 可以查询某个任务的关联资讯。
- [ ] 无权限任务不能查询关联资讯。
- [ ] 可以按任务标题刷新并关联资讯。
- [ ] 可以按手动关键词刷新并关联资讯。
- [ ] 重复刷新不会产生重复关联。
- [ ] 资讯刷新失败不影响任务详情查看和编辑。
- [ ] 前端任务详情弹窗能展示关联资讯列表。
