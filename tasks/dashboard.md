# 仪表盘

## 模块目标

实现当前用户权限范围内的任务统计、完成率计算、ECharts 图表和临期/逾期提醒展示。

## 输入依据

- `plan.md`：个人仪表盘展示待办数量、已完成数量和完成率图表。
- `plan.md`：建议加分功能包含统计图表和截止日期提醒。
- `plan.md`：仪表盘统计 API 包含 `/api/dashboard/summary` 和 `/api/dashboard/status-chart`。
- `detailed-design.md`：仪表盘模块包含导师全量统计、实习生本人统计、完成率计算、统计卡片和 ECharts 图表。

## 最小任务 checklist

- [ ] 定义仪表盘汇总响应 DTO，包含待办、进行中、已完成、总数和完成率。
- [ ] 定义状态分布图响应 DTO，适配 ECharts 使用。
- [ ] 仪表盘查询逻辑复用当前用户权限范围。
- [ ] 实现导师统计全量任务。
- [ ] 实现实习生统计本人任务。
- [ ] 实现 `TODO` 数量统计。
- [ ] 实现 `IN_PROGRESS` 数量统计。
- [ ] 实现 `DONE` 数量统计。
- [ ] 实现总任务数统计。
- [ ] 实现完成率计算，空数据时完成率为 0。
- [ ] 实现状态分布数据查询。
- [ ] 实现临期任务查询规则。
- [ ] 实现逾期任务查询规则：截止日期早于当前日期且状态不是 `DONE`。
- [ ] 实现 `GET /api/dashboard/summary` 汇总接口。
- [ ] 实现 `GET /api/dashboard/status-chart` 状态分布接口。
- [ ] 如需要展示提醒，补充临期/逾期任务接口或复用任务筛选接口。
- [ ] 前端定义仪表盘 TypeScript 类型。
- [ ] 前端封装仪表盘 API。
- [ ] 前端实现仪表盘页面路由。
- [ ] 前端实现待办、进行中、已完成、完成率统计卡片。
- [ ] 前端集成 ECharts。
- [ ] 前端实现状态分布图或完成率图。
- [ ] 前端实现临期任务提示。
- [ ] 前端实现逾期任务提示。
- [ ] 前端仪表盘加载失败时展示友好错误或空状态。
- [ ] 前端根据当前登录用户展示其权限范围内的统计结果。

## 验收 checklist

- [ ] 导师看到的是全量任务统计。
- [ ] 实习生看到的是本人任务统计。
- [ ] 待办、进行中、已完成数量正确。
- [ ] 完成率计算正确。
- [ ] 无任务时完成率为 0，页面不报错。
- [ ] ECharts 图表能正常渲染。
- [ ] 逾期任务只包含未完成且截止日期早于当前日期的任务。
- [ ] 前端统计卡片和图表数据一致。
