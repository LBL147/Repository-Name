# 任务管理

## 模块目标

实现任务创建、查看、编辑、删除、详情展示和列表展示能力，并在后端统一落实导师与实习生的数据权限范围。

## 输入依据

- `plan.md`：任务管理模块要求支持任务 CRUD。
- `plan.md`：导师可查看所有任务，实习生只能查看和维护分配给自己的任务。
- `plan.md`：任务字段包含标题、描述、负责人、优先级、截止日期和状态。
- `detailed-design.md`：任务管理模块包含任务 CRUD、导师/实习生数据范围控制、创建人与负责人维护。
- `detailed-design.md`：任务 API 包含 `/api/tasks`、`/api/tasks/{id}`、编辑和删除接口。
