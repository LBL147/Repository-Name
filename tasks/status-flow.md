# 状态流转

## 模块目标

实现任务状态在 `TODO`、`IN_PROGRESS`、`DONE` 之间的更新能力，并校验角色、任务归属和非法状态。

## 输入依据

- `plan.md`：状态流转模块要求支持待办、进行中、已完成三种状态。
- `plan.md`：用户可通过按钮或拖拽方式完成任务状态流转，MVP 先保证按钮切换。
- `plan.md`：状态流转顺序为 `TODO -> IN_PROGRESS -> DONE`，也允许在编辑弹窗中修正状态。
- `detailed-design.md`：状态流转模块包含任务归属和角色校验、状态按钮、状态标签、无权限操作禁用。
- `detailed-design.md`：状态更新 API 为 `PATCH /api/tasks/{id}/status`。
